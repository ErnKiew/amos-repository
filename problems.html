The AMOS Repository - Problems

All problems will be formatted as such: "Name (Source) - Difficulty"

CB --> Codebreaker, CF --> Codeforces

Some problems will have spoiler tags that may accidentally serve as hints to the question. We recommend at least thinking through the problem before pressing it.

Difficulty scale:

  1. Easy to the point that we hope everyone we've ever taught can solve this with no issues
  2. Trivial/easy to most
  3. Not hard to most
  4. Needs a good foundation, but not hard
  5. Even with a good foundation, requires effort and thinking
  6. Difficult, requires good understanding
  7. Very tough
  8. Lower bound of programming heads' capabilities
  9. Rough upper bound of programming heads' capabilities
  10. Extremely tough even for programming heads, perhaps need tips/a very long time to think
  11. Even programming heads also give up

  • nprime (CB) - 3.5/10 Sidenote: There's a particular fast algorithm you're meant to use that generates the answer in O(nlogn) time: Eratosthenes Sieve Sidenote 2: A tad hard to implement but honestly a bit overrated in difficulty if you know the algorithm
  • Explosion (CB) - 3.5/10 Sidenote: You must use pair/custom sorting. Go learn that beforehand. (ask the programming heads!)
  • Swap sort (CB) - 4.5/10 Sidenote: Think about how you can measure how close an arrangement is to being sorted, and how that changes with a swap.
  • Illuminate (CB) - 5/10 Sidenote: Solve subtasks first. Likely you will not be able to find the optimal answer first try!
